# SQL intro Now that we have a general understanding of databases, it's time to make use of our data! The **S**tructured **Q**uery **L**anguage (**SQL**) allows us to define, modify and read our data at will (think back to CRUD). Today, we'll focus only on the part of the language that allows us to ask questions of the data and receive an answer (querying). Let's see what that looks like!
# Anatomy of a SQL Query A query is defined by a block of text called a **statement**. This can be seen as a single "question" to the database, which requires a single answer. Each statement contains keywords that are part of the language, and column and tables names that identify where to get the data from. You can also add conditions that will refine the query (for ex.: sort or group data). The first keyword in each query to read data is **SELECT**, which tells the DB that we want to read. This keyword is followed by the names of the columns we want to query. The second keyword is **FROM**, followed by the names of the tables from which we want data. We terminate the statement with a semicolon.
# Example of a SELECT Query This is the most basic form of query, and looks something like this: `SELECT * FROM Person;` This query returns the value of every column (*) for every row, from the table called "Person". Let's try a basic SELECT query in real time!
# Group Exercise 1 1. Open our DB in SQLite Browser 2. Click on the "Execute SQL" Tab 3. Type the following query and strike F5 (or click ▶) ```sql SELECT * FROM FOOD_NAME; ``` You should see a table in the result pane below where you entered the query: ![Solution 1](framework/img/workshop/grp_ex_1.png)
# The WHERE Clause Our first example selects all the rows from a table, but what if we want to get only specific entries from the data. The WHERE clause can optionally follow the FROM clause, and allows us to do this. The WHERE clause looks like: **WHERE (condition)** A condition contains: 1. The name of the **column** whose values we want to test 2. The **operator** we want to test values by (for ex.: =, <, >, <=, >=, like, in) 3. The **operand**, or value we want the column to match
# Group Exercise 2 Let's refine our previous query by adding a WHERE clause: 1. Add a clause with the condition `FoodID = 5` and execute the query to select only the food with ID 5 2. Can you design a query to select those foods with IDs up to and including 10?
# Matching text with LIKE Our previous example allows us to refine our selection by comparing numeric values, but what if we want to search for some text? The **LIKE** operator exists for this reason! It's form is: `WHERE column_name LIKE "some text here"` 1. Use double quotes around the required text 2. Use **%** as a wild card to match anything * "apple%" matches a string **beginning** in "apple" * "%apple" matches a string **ending** in "apple" * "%apple%" matches a string containing "apple" **anywhere** * "apple" matches that string exactly
# Group Exercise 3 Take 5 minutes to: 1. Pick some keywords to search the FOOD_NAME table for 2. Experiment with selecting data by matching different strings 3. Experiment with the use of wildcards
# Ordering Results The results resturned by a query can be ordered using the `ORDER BY` clause: 1. Added following the `FROM` and `WHERE` clauses 2. Follows the form: `ORDER BY` **(columns) (ASC|DESC)**, where: * **(columns)** is one or more column names (separated by `,`) * **(ASC|DESC)** is `ASC` to sort in ascending order (small-to-large), or `DESC` to sort in descending order (large-to-small) Let's try that out!
# Exercise 4 Let's reuse our last query using `LIKE` to: 1. Sort elements by `FoodID`, from largest to smallest 2. Sort elements in ascending order in `FoodDescription` For example: ```sql SELECT FoodID, FoodGroupID, FoodSourceID, FoodDescription FROM FOOD_NAME WHERE FoodDescription LIKE "%apple%" ORDER BY FoodID DESC; ```
